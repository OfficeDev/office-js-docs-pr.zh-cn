# <a name="outlook-javascript-api-requirement-sets"></a><span data-ttu-id="71bee-101">Outlook JavaScript API 要求集</span><span class="sxs-lookup"><span data-stu-id="71bee-101">OneNote JavaScript API requirement sets</span></span>

<span data-ttu-id="71bee-p101">Outlook 加载项通过在其[清单](https://docs.microsoft.com/office/dev/add-ins/develop/add-in-manifests)中使用 [Requirements](/office/dev/add-ins/reference/manifest/requirements) 元素来声明所需要的 API 版本。Outlook 加载项始终包括 [Set](/office/dev/add-ins/reference/manifest/set) 元素，其中 `Name` 属性设置为 且 `Mailbox` 且 `MinVersion`  属性设置为支持加载项方案的 API 最低要求集。</span><span class="sxs-lookup"><span data-stu-id="71bee-p101">Outlook add-ins declare what API versions they require by using the [Requirements](/office/dev/add-ins/reference/manifest/requirements) element in their [manifest](https://docs.microsoft.com/office/dev/add-ins/develop/add-in-manifests). Outlook add-ins always include a [Set](/office/dev/add-ins/reference/manifest/set) element with a `Name` attribute set to `Mailbox` and a `MinVersion` attribute set to the minimum API requirement set that supports the add-in's scenarios.</span></span>

<span data-ttu-id="71bee-104">例如，下面的清单片段表示 1.1 的最低要求集：</span><span class="sxs-lookup"><span data-stu-id="71bee-104">For example, the following manifest snippet indicates a minimum requirement set of 1.1:</span></span>

```xml
<Requirements>
  <Sets>
    <Set Name="MailBox" MinVersion="1.1" />
  </Sets>
</Requirements>
```

<span data-ttu-id="71bee-p102">所有 Outlook API 都属于 `Mailbox` [ 要求集](https://docs.microsoft.com/office/dev/add-ins/develop/specify-office-hosts-and-api-requirements)。`Mailbox` 要求集具有多个版本，并且我们发布的每组新的 API 都属于更高版本的要求集。并非所有的 Outlook 客户端都支持最新的 API 集，但如果 Outlook 客户端声明支持要求集，则它支持该要求集中的所有 API。</span><span class="sxs-lookup"><span data-stu-id="71bee-p102">All Outlook APIs belong to the `Mailbox` [requirement set](https://docs.microsoft.com/office/dev/add-ins/develop/specify-office-hosts-and-api-requirements). The `Mailbox` requirement set has versions, and each new set of APIs that we release belongs to a higher version of the set. Not all Outlook clients support the newest set of APIs, but if an Outlook client declares support for a requirement set, it supports all of the APIs in that requirement set.</span></span>

<span data-ttu-id="71bee-p103">在清单中设置最低要求集版本可控制加载项会显示在哪个 Outlook 客户端中。如果客户端不支持最低要求集，则不会加载加载项。例如，如果指定要求集版本 1.3，则意味着加载项不会显示在任何不支持 1.3 及以上版本的 Outlook 客户端中。</span><span class="sxs-lookup"><span data-stu-id="71bee-p103">Setting a minimum requirement set version in the manifest controls which Outlook client the add-in will appear in. If a client does not support the minimum requirement set, it does not load the add-in. For example, if requirement set version 1.3 is specified, this means the add-in will not show up in any Outlook client that doesn't support at least 1.3.</span></span>

## <a name="using-apis-from-later-requirement-sets"></a><span data-ttu-id="71bee-111">使用更高版本要求集中的 API</span><span class="sxs-lookup"><span data-stu-id="71bee-111">Using APIs from later requirement sets</span></span>

<span data-ttu-id="71bee-p104">设置要求集不会限制加载项可使用的可用 API。例如，如果加载项指定要求集 1.1，但它在支持 1.3 的 Outlook 客户端中运行，则加载项可以使用要求集 1.3 中的 API。</span><span class="sxs-lookup"><span data-stu-id="71bee-p104">Setting a requirement set does not limit the available APIs that the add-in can use. For example, if the add-in specifies requirement set 1.1, but it is running in an Outlook client which support 1.3, the add-in can use APIs from requirement set 1.3.</span></span>

<span data-ttu-id="71bee-114">要使用较新的 API，开发人员可使用标准 JavaScript 技术来检查是否存在新 API：</span><span class="sxs-lookup"><span data-stu-id="71bee-114">To use newer APIs, developers can just check for their existence by using standard JavaScript technique</span></span>

```js
if (item.somePropertyOrFunction !== undefined) {
  item.somePropertyOrFunction ...
}
```

<span data-ttu-id="71bee-115">对于清单中所指定的要求集版本中的任何 API，无需执行此类检查。</span><span class="sxs-lookup"><span data-stu-id="71bee-115">No such checks are necessary for any APIs which are present in the requirement set version specified in in the manifest.</span></span>

## <a name="choosing-a-minimum-requirement-set"></a><span data-ttu-id="71bee-116">选择最低要求集</span><span class="sxs-lookup"><span data-stu-id="71bee-116">Choosing a minimum requirement set</span></span>

<span data-ttu-id="71bee-117">开发人员应使用包含其方案关键 API 集的最早要求集，如果不使用该要求集，外接程序将不起作用。</span><span class="sxs-lookup"><span data-stu-id="71bee-117">Developers should use the earliest requirement set that contains the critical set of APIs for their scenario, without which the add-in won't work.</span></span>
